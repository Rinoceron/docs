# Delinquency and Collateral

Delinquency is managed as controlled asset execution, not only communication follow-up.

## Trigger

- Missed payment
- `days_past_due > 0`

## Stage progression

`0_7 -> 8_15 -> 16_30 -> REPO`

## What changes in data

- `delinquency` tracks stage and case-level evolution
- `collateral.remaining` decreases when execution occurs
- `transaction` includes chargeoff/adjustment events during execution
- `comm_log` records escalation evidence

## Recovery modes

- Partial liquidation
- Full liquidation
- Communication escalation with legal-grade traceability
